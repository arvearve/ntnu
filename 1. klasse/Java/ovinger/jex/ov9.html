Part 1: Observable highscore list (75 pt.)

In this part you will implement an observable highscore list, by means of a class HighscoreList inheriting from an abstract class ObservableList. Observability is based on the listener interface ListListener.

ObservableList implements support for adding and removing ListListener objects, notifying these listeners about changes to a list, and adding and removing elements with proper notifications.

HighscoreList contains elements, e.g. game results, implementing the Comparable interface. By inheriting from ObservableList it supports listening to changes to the highscores.
public interface ListListener
The ListListener interface is implemented by classes that want to be notified about changes to an ObservableList. It contains one method, listChanged, that is called when the ObservableList changes.

public void method listChanged in ListListener
The method listChanged, that is called when the ObservableList changes. It has receives three arguments: the ObservableList that has changed, the lowest index that has changed and the highest index that has changed. Note that both indices can be greater or equal to the list's size, since they may refer to indices that existed before the change, but not after. E.g. if the last element of the ObservableList is removed, both indices will be equal to the list's size.

public abstract class ObservableList (50 pt.)
ObservableList implements support for adding and removing ListListener objects, notifying these listeners about changes to a list, and adding and removing elements with proper notifications.

public void method addListListener in ObservableList
The addListListener method adds the ListListener argument to the set of ListListeners that are notified when this ObservableList changes.

Duplicate listeners are not allowed and order must be preserved. Question: What kind of Collection should be used for holding the ListListeners, given these requirements?
public void method removeListListener in ObservableList
The removeListListener method removes the ListListener argument from the set of ListListeners that are notified when this ObservableList changes.

protected void method fireListChanged in ObservableList
The fireListChanged method takes two indices as argument, and notifies the ListListeners that this ObservableList has changed. The arguments correspond to those of ListListener.

This is a utility method that can be called from within ObservableList and from subclasses.
protected abstract List method getList in ObservableList
The getList method returns a non-specialized List. The list must be the actual list that is observed. The method is abstract and hence must be implemented by (non-abstract) subclasses. ObservableList must use this method in its implementation if the size, addElement and removeElement methods.

Question: Why should this method be protected and not private or public?
public int method size in ObservableList
The size method returns the size of this ObservableList.

protected void method addElement in ObservableList
The addElement method takes two arguments, an index and an element. The method adds the element at the provided index to this ObservableList and notifies the ListListeners about the change.

Take care to call the fireListChanged method with the correct indices, according to the requirements of the ListListener.listChanged method.
protected void method removeElement in ObservableList
The removeElement method removes the element at the provided index from this ObservableList and notifies the ListListeners about the change.

Take care to call the fireListChanged method with the correct indices, according to the requirements of the ListListener.listChanged method.
public class HighscoreList (25 pt.)
The HighscoreList class contains a limited number of elements, e.g. game results, implementing the Comparable interface (to enable sorting). It inherits much of its implementation from ObservableList, e.g. its support for ListListeners.

This class must implement the Iterable<Comparable> interface, to support the simplified for looping construct.

public constructor HighscoreList
HighscoreList's constructor must take an int argument, that specifies the maximum size of the list. If the list becomes too large, by calling addResult too many times, the extra elements (lowest rated) must be automatically removed.

public void method addResult in HighscoreList
The addResult method takes an element implementing the Comparable interface, and adds it to the list.

The element must be added at the appropriate position and the maximum size of the list must be considered.
public Iterator<Comparable> method iterator in HighscoreList
The iterator method implements the corresponding method from the Iterable<Comparable> interface.
